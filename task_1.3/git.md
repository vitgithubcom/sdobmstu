Команды  GIT

git init	Инициализирует новый репозиторий Git в текущей папке. С этого момента Git начнет отслеживать изменения в проекте .

git clone <url>	Создает копию удаленного репозитория (например, с GitHub) на вашем локальном компьютере .	
git clone https://github.com/пользователь/репозиторий.git

git status	Показывает текущее состояние рабочей директории и индекса. Отображает, какие файлы изменены, добавлены или не отслеживаются Git . Это одна из самых полезных команд для ориентации.

git add <файл>	Добавляет изменения из рабочего каталога в так называемую область подготовки (staging area). Файлы становятся подготовленными к коммиту .	
git add README.md (добавить один файл)

git add . (добавить все изменения в текущей папке)

git commit -m "сообщение"	Записывает изменения из области подготовки (staging area) в репозиторий, создавая коммит — снапшот состояния проекта на данный момент. Сообщение должно кратко и ясно описывать суть изменений .	
git commit -m "Добавлена форма обратной связи"

git log	Выводит историю коммитов в обратном хронологическом порядке. Можно увидеть хеш коммита, автора, дату и сообщение .	
git log

git log --oneline (сокращенный формат)

git push	Отправляет ваши локальные коммиты в удаленный репозиторий (например, на GitHub) .	
git push origin main

git pull	Забирает изменения из удаленного репозитория и сливает (мержит) их с вашей текущей локальной веткой . Перед началом работы всегда полезно выполнить git pull, чтобы получить актуальную версию проекта .	
git pull origin main
git pull github master

git remote -v   покажет, какие репозитории уже подключены к вашему проекту.
