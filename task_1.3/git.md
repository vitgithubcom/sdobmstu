# Команды Git

## Базовые команды для начала работы

### git init
Назначение: Инициализирует новый репозиторий Git в текущей папке.
**С этого момента Git начнет отслеживать изменения в проекте.**

---

### git clone <url>
Назначение: Создает копию удаленного репозитория (например, с GitHub) на вашем локальном компьютере.

Пример:
[git clone](https://github.com/пользователь/репозиторий.git)

---

### git status
Назначение: Показывает текущее состояние рабочей директории и индекса.
Отображает, какие файлы изменены, добавлены или не отслеживаются Git.

Совет: Это одна из самых полезных команд для ориентации.

---

## Работа с изменениями

### git add <файл>
Назначение: Добавляет изменения из рабочего каталога в так называемую область подготовки (staging area). Файлы становятся подготовленными к коммиту.

Примеры:
| Команда | Комментарий |
|-------------|-------------|
| git add README.md    | # добавить один файл    |
| git add . | # добавить все изменения в текущей папке |
             

---

### git commit -m "сообщение"
Назначение: Записывает изменения из области подготовки (staging area) в репозиторий, создавая коммит — снапшот состояния проекта на данный момент.

Пример:
git commit -m "Добавлена форма обратной связи"

> Важно: Сообщение должно кратко и ясно описывать суть изменений.

---

### git log
Назначение: Выводит историю коммитов в обратном хронологическом порядке. Можно увидеть хеш коммита, автора, дату и сообщение.

Примеры:
git log                  # полная история
git log --oneline        # сокращенный формат (одна строка на коммит)

---

### git branch
Назначение: Новая ветка.

Примеры:
git branch               # просмотр состояния веток
git branch newBranch     # создание новой ветки
git checkout newBranch   # перейти на новую ветку
git add .
git commit -m "qwert"
git branch               # просмотр состояния веток
git checkout master      # перейти в мастер
git merge newBranch      # присоединение новой ветки - нужно быть в мастере!
git branch -d newBranch  # удаление новой ветки

---

## Работа с удаленными репозиториями

### git pull
Назначение: Забирает изменения из удаленного репозитория и сливает (мержит) их с вашей текущей локальной веткой.

Пример:
git pull origin main

Совет: Перед началом работы всегда полезно выполнить git pull, чтобы получить актуальную версию проекта.

---

### git push
Назначение: Отправляет ваши локальные коммиты в удаленный репозиторий (например, на GitHub).

Примеры:
git push origin main
git push github master

---

### git remote -v
Назначение: Показывает, какие удаленные репозитории уже подключены к вашему проекту.

## <a id="shpargalka"></a> Краткая шпаргалка

- `git init` — Создать новый репозиторий
- `git clone <url>` — Скопировать удаленный репозиторий
- `git status` — Проверить состояние
- `git add <файл>` — Добавить файлы в staging area
- `git commit -m "..."` — Создать коммит
- `git log` — Посмотреть историю
- `git branch` — Ветка
- `git merge` — Объединить ветки
- `git pull` — Получить изменения из удаленного репозитория
- `git push` — Отправить изменения в удаленный репозиторий
- `git remote -v` — Посмотреть подключенные удаленные репозитории
